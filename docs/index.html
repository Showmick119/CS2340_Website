Built on Solid Foundations: MVVM Architecture

SpendWise implements the Model-View-ViewModel (MVVM) architectural pattern throughout all four sprints, ensuring clean separation of concerns, testability, and maintainability.

View Layer (UI)

Activities

MainActivity, Login, Register, Dashboard, ExpenseLog, Budgetlog, SavingCircleLog, SavingCircleDetailActivity, InvitationsActivity, Chatbot, BudgetDetails

Adapters

BudgetAdapter, ExpenseAdapter, SavingCircleAdapter, InvitationAdapter, NotificationAdapter

Responsibility

UI rendering and user interaction capture. Observes LiveData from ViewModels. No business logic or Firebase calls.

ViewModel Layer (Business Logic)

ViewModels

LoginViewModel, ExpenseViewModel, BudgetViewModel, SavingCircleViewModel, DashboardAnalyticsViewModel, ChatbotViewModel, NotificationViewModel

Responsibility

Handle all business logic, data transformation, validation, and coordinate between Views and Models. Expose LiveData observables. Survive configuration changes.

Model Layer (Data & Domain)

Domain Models

User, Expense, Budget, Category, SavingCircle, SavingCircleMember, MemberCycle, SavingCircleInvitation, ChatMessage, ChatSession, BudgetUsageSummary

Infrastructure

AnalyticsRepository, Firebase Singleton (thread-safe double-checked locking)

Responsibility

Domain entities with proper encapsulation. Repository aggregates complex operations. All Firebase Realtime Database interactions mediated through Singleton.

Design Patterns Implemented

üîí Singleton Pattern

Class: Firebase.java

Implementation: Thread-safe double-checked locking with volatile keywords

Purpose: Ensures single instance of Firebase Database connection across the application, preventing resource wastage and ensuring data consistency.

üè≠ Factory Pattern

Class: ChartFactory.java

Methods: createCategoryPieChart(), createBudgetBarChart()

Purpose: Centralizes chart object creation logic, abstracting MPAndroidChart configuration complexity from Views. Allows easy addition of new chart types.

‚ö° Strategy Pattern

Interface: ExpenseSortStrategy

Implementations: SortByDateStrategy, SortByAmountStrategy, SortByCategoryStrategy

Purpose: Allows runtime selection of different sorting algorithms. New sorting methods can be added without modifying existing code.

SOLID Principles Application

S
Single Responsibility

Each ViewModel handles one feature domain. Models contain only data and domain logic.

O
Open/Closed

Strategy pattern allows extension of sorting behaviors without modifying ExpenseViewModel code.

L
Liskov Substitution

All strategy implementations can substitute for ExpenseSortStrategy interface without breaking functionality.

I
Interface Segregation

Focused interfaces like ExpenseSortStrategy ensure classes only depend on methods they use.

D
Dependency Inversion

ViewModels depend on Repository abstractions rather than concrete Firebase implementation.

Feature-Rich Financial Management

01

Authentication System

  • Welcome Screen: Clean splash screen with Start and Quit buttons
  • Login Screen: Email and password with validation (whitespace/null checks)
  • Registration: New account creation with Firebase Authentication
  • Security: All credentials stored securely with automatic session management
02

Dashboard - Financial Command Center

  • Overview Cards: Total spent and remaining amounts with color-coded indicators
  • Interactive Charts: Pie chart for expense breakdown by category, bar chart for budget usage
  • Quick Budgets List: Scrollable RecyclerView with status badges (green/yellow/red)
  • Calendar Selector: Date simulation for testing time-based features
  • Real-time Updates: Charts update immediately using LiveData observers
03

Expense Log

  • Expense List: Sorted by date with name, amount, category, date visible
  • Add Button (FAB): Floating action button for expense creation
  • Creation Form: Name, Amount (>0), Category dropdown, Date picker (‚â§ current date), Notes
  • Savings Circle Integration: Link expenses to group challenges
  • Validation: Real-time error messages with focus management
04

Budget Management

  • Budget List: Sorted by start date with visual progress bars
  • Creation Form: Title, Amount, Category, Frequency (Weekly/Monthly), Start Date
  • Budget Details: Progress indicator, surplus/deficit calculation, linked expenses
  • Budget Calculator: 2-of-3 system (Total/Spent/Remaining)
  • Automatic Rollover: Budgets reset based on frequency and simulated date
05

Savings Circles - Group Challenges

  • Create Circle: Group name, challenge title, goal amount, frequency, personal allocation
  • Invitation System: Send invites by email, accept/decline in InvitationsActivity
  • Circle Details: Member information, individual contributions, group progress
  • Real-time Sync: All members see updates immediately via Firebase listeners
  • Cycle Management: Weekly (7 days) or Monthly periods from acceptance date
06

AI Chatbot - Financial Advisor

  • AI Model: Local Ollama server running LLaMA 3.2
  • Custom Commands: "Spending this week", "Cut costs", "My budget", "My expenses"
  • Context-Aware: Fetches real user data from Firebase for personalized advice
  • Session Persistence: All conversations saved with AI-generated titles
  • Previous Chats: Dialog showing all past sessions, load on tap
07

Smart Notifications System

  • Missed Expense Alert: Triggered after 3 days without logging
  • Budget Threshold Warning: Alert at 90% utilization
  • Queue System: Prevents notification overlap
  • Custom Dialog: RecyclerView showing all pending alerts
  • Persistence: Dismissed notifications stored in Firebase to prevent re-showing
